# Data Asset Map

本文档用于说明仓库中不同数据目录的角色，帮助区分：

- 哪些数据是历史导出
- 哪些数据是中间阶段使用的
- 哪些数据是当前主线真正使用的

---

## 1. 为什么需要这份文档

本仓库经历了多个开发阶段，因此数据文件也在多个目录中留下了痕迹。  
常见内容包括：

- `input_*.hex`
- `pred_*.txt`
- `preds.txt`
- `quant_params.hex`
- `fc1_weight_int8.hex`
- `fc1_bias_int32.hex`
- `fc2_weight_int8.hex`
- `fc2_bias_int32.hex`
- `route_b_output*`

如果不专门说明，很容易出现以下困惑：

- 不知道哪个 `.hex` 是当前实际在用的
- 不知道哪个 `route_b_output` 是历史备份
- 不知道板级工程到底依赖哪个数据目录

---

## 2. 当前唯一推荐使用的数据入口

当前主线工程真正使用的数据目录是：

```text
FPGA_A/data/
```

这是后续仿真、综合、实现和上板时，默认应当依赖的数据入口。

其子目录通常包括：

- `FPGA_A/data/samples/`
- `FPGA_A/data/weights/`
- `FPGA_A/data/quant/`
- `FPGA_A/data/expected/`

### 含义
#### `samples/`
样本输入数据，例如：

- `mnist_samples_route_b_output_2.hex`
- `input_0.hex`
- `input_1.hex`
- ...

#### `weights/`
网络参数，例如：

- `fc1_weight_int8.hex`
- `fc1_bias_int32.hex`
- `fc2_weight_int8.hex`
- `fc2_bias_int32.hex`

#### `quant/`
量化参数，例如：

- `quant_params.hex`
- `quant_config.json`

#### `expected/`
参考输出，例如：

- `pred_0.txt`
- `pred_1.txt`
- `preds.txt`
- `labels.txt`

## 3. 历史数据目录的角色

## 3.1 `hw/` 中的数据
`hw/` 目录下通常保留了研究阶段或功能验证阶段的完整导出结果。

这些数据的特点是：

- 更完整
- 更接近原始实验产物
- 适合做历史回溯
- 不一定是当前板级工程直接引用的版本

常见用途：

- 查看最初导出的多组结果
- 回溯某次实验的输入/输出
- 对照架构验证时期的功能结果

---

## 3.2 `hw_A/` 中的数据
`hw_A/` 中的数据通常与“方案 A 第一次工程化整理”相关。

这些数据的特点是：

- 更接近上板场景
- 与 UART 接入后的工程组织方式更相关
- 可能是 `FPGA_A/data/` 的上游来源之一

适合用来：

- 回溯板级工程早期使用的数据形式
- 比较中间阶段与最终阶段的数据组织方式

---

## 3.3 `onboard_A/` 中的数据
`onboard_A/` 中的数据处于进一步收敛阶段。

这些数据的特点是：

- 结构更清晰
- 更面向板级工程
- 往往已经接近最终 `FPGA_A/data/` 的形态

适合用来：

- 回看板级收敛中间状态
- 对照 `FPGA_A/data/` 的形成过程

---

## 3.4 `sw/` 中的数据
`sw/` 中的数据更接近数据源头。

这些数据的特点是：

- 来自训练、量化和导出流程
- 是 `.hex` 文件的上游来源
- 更适合追溯“这些参数和样本是怎么生成出来的”

如果你想回答：

> “这些硬件用到的 `.hex` 到底是从哪里来的？”

那应优先看 `sw/`。

---

## 4. 如何判断当前应使用哪个数据目录

原则很简单：

### 板级仿真 / 综合 / 上板
使用：

```text
FPGA_A/data/
```

### 历史回溯 / 对拍 / 查来源
再去看：

- `onboard_A/data/`
- `hw_A/data/`
- `hw/.../route_b_output*`
- `sw/...`

---

## 5. 推荐使用规则

为了避免混乱，建议以后遵守下面这条规则：

> 所有当前主线使用的数据，以 `FPGA_A/data/` 为准。

这意味着：

- 仿真脚本优先引用 `FPGA_A/data/`
- Vivado 工程优先加入 `FPGA_A/data/`
- README 和使用文档优先写 `FPGA_A/data/`
- 其他历史目录只作为参考和追溯来源

---

## 6. 常见问题

### Q1：为什么同样的 `fc1_weight_int8.hex` 在多个目录里都出现？
因为项目经历了多个开发阶段，每个阶段都保留了自己的可运行快照。  
这些文件名可能相同，但它们所处的目录阶段不同，含义也不同。

### Q2：我现在应该用哪个版本？
默认用：

```text
FPGA_A/data/
```

### Q3：那历史目录中的数据还有必要保留吗？
有必要。  
它们对以下事情有价值：

- 复盘开发过程
- 回溯某次实验
- 支持答辩或论文中描述版本演进
- 查找某个数据第一次出现于哪个阶段

---

## 7. 一句话总结

当前主线数据入口只有一个：

```text
FPGA_A/data/
```

其他目录中的数据保留用于历史回溯、来源追踪和阶段对照，而不是当前主线的默认依赖。
