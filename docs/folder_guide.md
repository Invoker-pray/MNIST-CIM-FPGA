# Folder Guide

本文档用于回答一个最实际的问题：

> 如果我现在要做某件事，我应该进入哪个目录？

---

## 1. 总体原则

当前仓库中有多个历史阶段目录，但**唯一推荐继续开发的主线是**：

```text
FPGA_A/
```

其他目录保留用于参考、回溯和说明开发历程，不建议继续作为主线修改。

---

## 2. 如果你想做某件事，该去哪里

## 2.1 我想继续开发最终上板工程
进入：

```text
FPGA_A/
```

这是当前主线目录，适合进行：

- 修 RTL bug
- 调整板级逻辑
- 修改约束文件
- 重新综合实现
- 上板调试

---

## 2.2 我想看最终上板 RTL
进入：

```text
FPGA_A/rtl/
```

这里包含：

- `pkg/`
- `ctrl/`
- `mem/`
- `core/`
- `uart/`
- `top/`

是当前最终设计代码所在位置。

---

## 2.3 我想运行仿真
进入：

```text
FPGA_A/scripts/
```

查看或执行：

- `run.sh`
- 其他仿真相关脚本

对应 testbench 通常在：

```text
FPGA_A/tb/
```

---

## 2.4 我想构建 Vivado 工程并生成 bitstream
进入：

```text
FPGA_A/scripts/
```

重点关注：

- `vivado_build.sh`
- `vivado_build.tcl`

这些脚本会在 batch 模式下创建并构建 Vivado 工程。

---

## 2.5 我想看板级约束
进入：

```text
FPGA_A/constr/
```

重点文件：

```text
top.xdc
```

这里定义了：

- 时钟引脚
- 按键
- 样本选择输入
- UART TX
- LED 状态输出

---

## 2.6 我想看当前最终使用的数据文件
进入：

```text
FPGA_A/data/
```

按用途分为：

- `samples/`
- `weights/`
- `quant/`
- `expected/`

这里是当前主线工程真正使用的数据入口。

---

## 2.7 我想看最早的研究原型
进入：

```text
hw/
```

这个目录适合用来：

- 查看最初流程设计
- 查看 tile 计算原型
- 查看并行结构验证
- 查看 shared buffer 方案最初实现

---

## 2.8 我想看 UART 是怎么开始接进来的
进入：

```text
hw_A/
```

这个目录更适合查看：

- UART 初始接入版本
- 更早期的工程化整理
- 更适合上板的 `rtl_ip` 封装起点

---

## 2.9 我想看上板收敛过程中的中间版本
进入：

```text
onboard_A/
```

这个目录适合查看：

- 文件结构进一步重整的过程
- 消抖 / done latch / 单脉冲这类板级逻辑的收敛版本
- 从过渡版走向最终主线之前的工程状态

---

## 2.10 我想看软件训练与量化导出流程
进入：

```text
sw/
```

这里适合查看：

- 软件训练逻辑
- 量化导出流程
- `.hex` 等硬件初始化数据的来源

---

## 2.11 我想看开发记录、过程讨论和辅助文档
进入：

```text
AI-used/
```

这个目录更适合查看开发过程中的记录、思考和中间文档。

## 4. 推荐阅读路径

如果你是第一次接触本仓库，建议按这个顺序看：

1. `README.md`
2. `docs/project_timeline.md`
3. `docs/data_asset_map.md`
4. `FPGA_A/`
5. 如果需要回溯，再看 `onboard_A / hw_A / hw`

---

## 5. 当前维护建议

为了避免后续继续混乱，建议遵守以下规则：

- 新功能只加到 `FPGA_A`
- 历史目录不再作为主线继续修改
- 文档说明优先放到 `docs/`
- 数据的当前有效版本以 `FPGA_A/data/` 为准

---

## 6. 一句话总结

如果你不确定去哪，默认先去：

```text
FPGA_A/
```

因为它是当前唯一推荐继续开发、综合、实现和上板的主线目录。
