# FPGA_A 上板使用指南

## 1. 目标

本工程完成后，FPGA 板上的交互目标是：

- 通过板上输入选择 `sample_sel`
- 按下启动键 `btn_start` 发起一次推理
- 通过 LED 观察运行状态
- 通过 UART 串口读取最终预测结果

当前系统不是“边算边打印完整中间张量”的交互式调试器，而是一个**板级推理演示系统**：

- 输入：板上按键 / 拨码 / PMOD 输入
- 输出：LED 状态 + UART 预测结果

---

## 2. 上板后你能看到什么

### 2.1 输入侧

你当前顶层支持的输入包括：

- `rst_n`：复位
- `btn_start`：启动一次推理
- `sample_sel[4:0]`：选择样本编号

### 2.2 输出侧

你当前顶层支持的输出包括：

- `led_busy`：表示推理正在进行
- `led_done`：表示一次推理已经完成（锁存保持）
- `uart_tx`：串口发送预测结果

---

## 3. 上板前准备

在烧录 bitstream 之前，请确认以下事项已经完成：

- Vivado 综合、实现和 bitstream 生成成功
- `top.xdc` 已正确绑定板卡引脚
- 板卡供电正常
- USB-JTAG 已连接到电脑
- 如果要看 UART 输出，已经准备好串口接收方式

---

## 4. 烧录 bitstream

你可以在 Vivado 里用 Hardware Manager 下载，也可以用你后续准备的 batch 下载脚本。

烧录成功后，FPGA 会开始按照当前设计工作。

---

## 5. 板上交互方式

## 5.1 复位

`rst_n` 是低有效复位。

如果你在约束中把 `rst_n` 绑定到了拨码开关，例如 `SW1`，则：

- `SW1 = 1`：系统正常运行
- `SW1 = 0`：系统复位

建议上电后先保持复位，再释放复位开始测试。

---

## 5.2 选择样本

`sample_sel[4:0]` 决定本次推理使用哪个样本。

你当前工程中，`sample_sel` 由板上输入组合而成，例如：

- 若干按钮
- 若干拨码
- 一个 PMOD 输入位

这 5 位组合在一起表示一个样本编号：

- `00000` 表示 sample 0
- `00001` 表示 sample 1
- `00010` 表示 sample 2
- ...
- `10011` 表示 sample 19

你应当在按下启动键之前，先把 `sample_sel` 设置好。

---

## 5.3 启动一次推理

按下 `btn_start` 后，系统会：

1. 对按键做消抖
2. 生成一个单周期 `start_pulse`
3. 启动一次完整推理

注意：

- 按住不放不会持续重复触发
- 只有一次有效上升沿会触发一次推理

---

## 6. 如何观察计算过程

## 6.1 通过 LED 观察系统状态

### `led_busy`

当推理开始后，`led_busy` 会亮起或置高，表示系统正在计算。

这意味着内部正在执行：

- 样本读取
- FC1 分块推理
- ReLU / requantize
- FC2 推理
- argmax 分类

### `led_done`

当一次推理完成后，`led_done` 会亮起并保持。

你已经加入了 `done latch`，所以这个灯不会只闪一下，而是会保持，直到：

- 重新复位
- 或者下一次新的 `start_pulse` 到来

这使得你在板上能明确知道“这一轮推理已经结束”。

---

## 6.2 当前版本不能直接看到的内容

当前系统默认**不会在板上逐步打印中间计算结果**，例如：

- 每个 FC1 block 的部分和
- ReLU 后的 128 维中间向量
- FC2 的全部 logits

这些中间量目前主要在仿真中观察，而不是在板上直接输出。

板上当前的“计算过程可见性”主要通过：

- `led_busy`
- `led_done`
- UART 最终预测结果

来体现。

---

## 7. 如何看到返回信息

## 7.1 返回信息的形式

当前设计通过 `uart_tx` 输出**最终预测类别**。

通常输出形式类似：

```text
7
```
